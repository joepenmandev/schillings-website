# Schillings — Final visual composition prompts

**Institutional evidence system (refined)**

These prompts are **compositionally intelligent**: they describe how compositions should behave, what emotional signal they create, what dominates, what disappears, and how a viewer should subconsciously evaluate the page — not merely what should appear.

Use them for image generation, designer handoff, or evidence-plate curation. They do not replace `IMAGE-SELECTION.md` or `VISUAL-EDITORIAL.md`; they operationalise them for production.

---

## Global system prompt

Use before **all** visual generation or composition.

Create restrained institutional evidence compositions for a premium legal and advisory firm briefing.

The compositions should feel:

- editorial
- architectural
- forensic
- institutionally composed
- commercially intelligent
- quietly premium

**Not:**

- promotional
- glossy
- startup-like
- luxury cliché
- marketing-oriented
- UI showcase driven

**Visual language should prioritise:**

- negative space
- compositional restraint
- selective interface fragments
- typography rhythm
- asymmetry
- muted institutional tones
- documentary calmness

**The viewer should feel:** they are examining institutional signals, not reviewing a website redesign.

**Do not create:**

- full-page browser screenshots
- device mockups
- glowing UI
- gradients
- visual theatre
- obvious “before vs after” comparisons
- oversized annotations
- presentation-deck aesthetics

**All visuals should behave like:** curated evidence extracted from a premium institutional review process.

## Compositional behavior rules

All compositions should obey the following editorial behavior rules:

1. One dominant visual idea only. Do not create visually equal competing regions.
2. Asymmetry is preferred over symmetry. Compositions should feel editorial and curated rather than presentation-grid based.
3. Negative space must remain active. Do not fill empty regions unnecessarily; whitespace should feel intentional and institutional.
4. Fragments should feel extracted rather than assembled. Interface regions should appear selectively observed rather than fully presented.
5. Typography should carry authority. Large interface structures are less important than hierarchy, rhythm and wording.
6. Crops should avoid complete visibility. Partial visibility creates sophistication and evaluative tension.
7. Archival elements should remain subordinate. Historical fragments should never compete visually with the revised institutional state.
8. Interface fragments should feel calm at distance. Avoid dense interaction patterns, visual clutter or excessive UI complexity.
9. Visual pacing should slow the reader down. Compositions should encourage observation rather than scanning.
10. Compositions should feel authored, not generated. Avoid over-balanced layouts, obvious AI symmetry or decorative sophistication.

## Visual failure states

Reject compositions that feel:

- like website screenshots
- like SaaS landing pages
- like agency case studies
- like product marketing
- like UI showcases
- like pitch decks
- like luxury branding cliches
- over-annotated
- over-symmetrical
- over-filled
- overly polished
- excessively layered
- too glossy
- too presentation-led

Reject any composition where:

- the interface becomes more important than the institutional signal
- visual activity overwhelms composure
- decorative styling replaces restraint
- browser or UI mechanics dominate perception
- hierarchy feels mechanically generated

## Visual priority hierarchy

Visual importance hierarchy:

1. Institutional composure
2. Typography rhythm
3. Spatial restraint
4. Hierarchy clarity
5. Fragment selection
6. Interface structure
7. Decorative polish

If decorative sophistication competes with composure, remove it.

## Scale and dominance guidance

Visuals should feel confidently scaled.

Preferred behavior:

- oversized crops
- partial visibility
- edge bleed
- dominant hierarchy moments
- restrained fragmentation

Avoid:

- timid centered screenshots
- overly framed interface regions
- excessive safe margins
- miniature UI compositions
- presentation-board scaling

Compositions should feel editorially cropped rather than carefully fitted.

## Transitional pacing rules

The briefing should feel progressively more resolved as it advances.

Page transitions should obey the following pacing behaviour:

1. **Page 1 → Page 2**  
   Move from institutional abstraction toward human credibility. The visual field should become slightly more intimate and evaluative.

2. **Page 2 → Page 3**  
   Move from individual authority toward institutional precision. Compositions should become more structured and systematic.

3. **Page 3 → Page 4**  
   Move from structural clarity toward institutional coherence. Fragment relationships and alignment become more important than individual elements.

4. **Page 4 → Page 5**  
   Move from coherence toward progression and engagement confidence. Operational clarity should feel calmer and more inevitable than process-driven.

5. **Page 5 → Page 6**  
   Move from operational confidence toward institutional resolution. The final composition should feel quieter, broader and more conclusive than previous spreads.

The briefing should feel increasingly composed, increasingly coherent, and increasingly inevitable as it progresses.

Avoid abrupt visual resets between spreads.

## Visual density governance

Visual density should remain restrained throughout the briefing.

**Rules:**

- No composition should contain more than one dominant focal hierarchy.
- Secondary fragments should remain subordinate.
- Layering should create depth, not clutter.
- The number of visible interface elements should remain intentionally limited.
- Visual complexity should never increase faster than narrative confidence.

If a composition begins feeling busy, designed or visually performative, reduce elements immediately.

The briefing should imply institutional confidence through omission.

## Fragment behavior rules

Fragments should behave like extracted institutional observations.

**Fragments should:**

- appear partially observed
- avoid complete interface exposure
- imply larger systems beyond the crop
- preserve ambiguity where appropriate
- prioritise hierarchy over completeness

**Fragments should not:**

- explain the entire interface
- function as screenshots
- expose full navigational systems
- appear presentation-assembled
- compete equally for attention

The viewer should feel they are observing selected institutional signals, not reviewing interface coverage.

## Typographic emphasis hierarchy

Typography should function as a primary institutional signal.

**Priority order:**

1. Hierarchy rhythm
2. Spatial restraint
3. Wording precision
4. Alignment consistency
5. Typeface styling

Typography should feel editorial, measured, and institutionally composed.

**Avoid:**

- oversized typographic drama
- decorative scaling
- aggressive emphasis
- excessive weight contrast
- theatrical typography behaviour

Authority should come from restraint and precision, not visual performance.

## Silence and restraint rules

Some regions of each composition should remain intentionally unresolved.

**Allow:**

- quiet negative space
- incomplete visibility
- subdued hierarchy regions
- low-information zones
- restrained visual pauses

Do not attempt to activate every region of the composition.

Silence is part of the authority signal.

The briefing should feel confident enough not to explain everything.
