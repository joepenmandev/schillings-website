# Objective

This briefing exists to communicate how the platform structure supports:

- authority
- trust
- enquiry quality
- referral confidence
- institutional positioning
- premium mandate acquisition

It does not exist to:

- showcase redesign work
- explain implementation
- present governance systems
- demonstrate SEO sophistication
- provide project chronology

# Required tone

The document must feel:

- institutional
- commercially intelligent
- calm
- premium
- externally forwardable
- strategically credible

It must not feel:

- agency-like
- promotional
- self-congratulatory
- technical
- operational
- startup or product-marketing-like
- deck-like

# Structural locks

Lock the following:

- 6-page structure
- image-led editorial spreads
- one strategic paragraph per spread
- one dominant visual anchor per spread
- restrained typography
- minimal annotation behavior
- no comparison-table logic
- no implementation storytelling

# Language locks

Avoid:

- redesigned
- improved
- optimized
- rollout
- implementation
- UX
- SEO
- governance
- framework
- architecture
- metadata

Prefer:

- clarified
- structured
- aligned
- positioned
- reinforced
- represented

# Process locks

- Narrative approval before visuals.
- Visual approval before composition.
- Composition is editorial assembly, not strategic exploration.

# Definition of done

The briefing is complete when:

- the commercial thesis is clear
- each spread has a distinct realization
- the structure's commercial rationale is understandable
- screenshots support the narrative rather than dominate it
- the document feels institutional and externally forwardable
- it no longer reads like a website project presentation
