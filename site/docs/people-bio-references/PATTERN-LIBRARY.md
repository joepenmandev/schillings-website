## 1) First-Screen Authority Patterns

### Pattern: Structured Identity Before Story
- **What it does:** Places role, seniority, and capability taxonomy before narrative biography.
- **Why it builds trust:** Evaluators can quickly verify "who this person is" before interpreting claims.
- **Where it appeared:** Blackstone directory, Lazard leadership, Evercore leadership, FT columnists.
- **Whether it fits Schillings:** Yes (high fit).
- **Implementation risk:** Over-compression can feel sterile if no human context follows.
- **Example application to Schillings:** Ensure each profile opens with clear role/practice context and factual scope markers before longer prose.

### Pattern: Independent Signal in First View
- **What it does:** Surfaces third-party or institutional credibility marker at top of profile flow.
- **Why it builds trust:** Reduces reliance on self-assertion.
- **Where it appeared:** Fountain Court (directory quote high on page), Essex (ranked recognition statements).
- **Whether it fits Schillings:** Yes (high fit).
- **Implementation risk:** If overused, can read like badge stacking.
- **Example application to Schillings:** Introduce one restrained external validation cue early, then move into factual bio.

## 2) Portrait Patterns

- **Whether it fits Schillings:** Yes (high fit).

- **Example application to Schillings:** Keep portrait as stable identity anchor while proof and role signals remain textual primary.

- **Whether it fits Schillings:** Yes (high fit).

## 3) Recognition / Proof Patterns

### Pattern: Proof Adjacent to Claim
- **What it does:** Places evidence directly near capability assertions.
- **Why it builds trust:** Limits interpretive gap between statement and substantiation.
- **Where it appeared:** Fountain Court, Essex, Economist contributor bios.
- **Whether it fits Schillings:** Yes (high fit).
- **Implementation risk:** Overly dense adjacency can hurt readability if every sentence carries evidence.
- **Example application to Schillings:** Pair major authority statements with one immediate concrete marker (recognition, role, or output).

### Pattern: Mixed Proof Stack
- **What it does:** Uses multiple proof types (independent directories, appointments, roles, outputs).
- **Why it builds trust:** Multi-source evidence appears more robust than single-channel validation.
- **Where it appeared:** Essex, Fountain Court, Brunswick, Economist profiles.
- **Whether it fits Schillings:** Yes (high fit).
- **Implementation risk:** Can become a laundry list without hierarchy.
- **Example application to Schillings:** Sequence proofs by evaluator relevance: role scope -> independent validation -> representative work -> intellectual output.

## 4) Biography Rhythm Patterns

- **What it does:** Moves from current remit, to operating context, to proof-rich examples.

- **Where it appeared:** Essex, Brunswick, Economist (Henry Curr, Simon Cox).
- **Whether it fits Schillings:** Yes (high fit).

- **Example application to Schillings:** Structure bios as remit -> real-world context -> evidence-backed capability.

### Pattern: Paragraph Function Discipline
- **What it does:** Ensures each paragraph has one distinct job (scope, proof, context, output).
- **Why it builds trust:** Prevents redundant adjective cycles and increases scan confidence.
- **Where it appeared:** Brunswick and Economist profiles (compact, functionally distinct prose).
- **Whether it fits Schillings:** Yes (high fit).
- **Implementation risk:** Over-editing can flatten nuance.
- **Example application to Schillings:** Audit each paragraph for unique information contribution before publication.

## 10) Patterns To Avoid

### Pattern to Avoid: Assertion Without Nearby Proof
- **What it does:** Makes strong capability claims with delayed or absent substantiation.
- **Why it weakens trust:** Sophisticated evaluators discount unsupported authority language.
- **Where it appeared:** Most visible risk in broad directory-level systems when profile depth is deferred.
- **Whether it fits Schillings:** No.
- **Implementation risk:** High; creates credibility drag quickly.
- **Example application to Schillings:** Do not use "leading/expert/top-tier" framing unless immediately grounded in specific evidence.

### Pattern to Avoid: Overlong Third-Party Quote Dumps
- **What it does:** Uses very long quote stacks as primary authority mechanism.
- **Why it weakens trust:** Can feel noisy, repetitive, and overly performative.
- **Where it appeared:** Long recommendation blocks in chambers profiles.
- **Whether it fits Schillings:** Partial at best.
- **Implementation risk:** Medium-high (signal dilution).
- **Example application to Schillings:** Curate fewer, higher-signal external references with clear relevance.

### Pattern to Avoid: Taxonomy Overload Without Decision Help
- **What it does:** Presents many categories without clear prioritization for evaluators.
- **Why it weakens trust:** Creates complexity without improving confidence.
- **Where it appeared:** Enterprise-scale people systems with large filter sets.
- **Whether it fits Schillings:** No (if uncurated).
- **Implementation risk:** Medium.
- **Example application to Schillings:** Keep capability taxonomy concise and evaluator-oriented, not exhaustive by default.

### Pattern to Avoid: Heroic Personal Branding Drift
- **What it does:** Shifts from institutional advisory voice to personality-led storytelling.
- **Why it weakens trust:** Can undermine perception of discretion and seriousness.
- **Where it appeared:** Potential risk in high-visibility executive profiles if transplanted directly.
- **Whether it fits Schillings:** No.
- **Implementation risk:** Medium-high.
- **Example application to Schillings:** Keep individual distinction factual and role-based rather than persona-forward.
