# Practitioner Bio Reference Research System

This directory is a research-only system for improving practitioner bios through structured benchmarking.

## Scope

- Study live external profiles from elite institutions.
- Extract recurring authority and trust patterns.
- Translate observations into reusable writing and structure principles.
- Keep Schillings implementation unchanged at this stage.

## Non-Goals

- No design rollout.
- No component or template changes.
- No page-level rewrites.
- No direct copying of external profile language.

## Required Lens

All reference analysis should be assessed through:

- trust accumulation
- authority density
- proof systems
- portrait treatment
- biography rhythm
- operational seriousness
- contact pathway restraint
- institutional tone

## File Map

- `REFERENCE-SOURCES.md`  
  Canonical list of benchmark sources by category.
- `REFERENCE-AUDIT-TEMPLATE.md`  
  Standard worksheet used to audit each source profile.
- `PATTERN-LIBRARY.md`  
  Pattern inventory with confidence ratings and anti-patterns.
- `BIO-IMPROVEMENT-PRINCIPLES.md`  
  Distilled principles for future Schillings copy improvement.
- `SCHILLINGS-BIO-GAP-AUDIT.md`  
  Current-state gap analysis against validated patterns.
- `BENCHMARK-BIO-PLAN.md`  
  Sequenced plan for future testing and rollout decisions.

## Working Method

1. Add and maintain benchmark source set.
2. Audit each source with the shared template.
3. Aggregate repeated patterns in the library.
4. Distill principles from high-confidence patterns only.
5. Compare Schillings current bios against those principles.
6. Propose a future test plan without implementation.

## Evidence Standard

- Prefer observable evidence (placement, phrasing type, hierarchy behavior).
- Record confidence level per pattern:
  - **High**: appears across multiple categories.
  - **Medium**: appears in multiple sources within one category.
  - **Low**: appears in a single source or context.
- Separate signal from style preference.

## Governance

- Keep this system factual, concise, and implementation-neutral.
- Document assumptions explicitly.
- Timestamp updates and maintain source traceability.
