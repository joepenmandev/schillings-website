# Governance Change Control

Purpose: formalize governance for the practitioner bio system so quality improvements persist without aesthetic drift.

## Governance Intent

This system prioritizes:

- trust-system thinking
- consistency over novelty
- evidence over aesthetics
- scalable institutional behavior

## 1) Governed Artifacts

The following are controlled doctrine documents:

- `BIO-IMPROVEMENT-PRINCIPLES.md`
- `PATTERN-LIBRARY.md`
- `SCHILLINGS-BIO-GAP-AUDIT.md`
- `PHASE-1-IMPLEMENTATION-SCOPE.md`
- `LIVE-BIO-ROLLOUT-PLAN.md`
- `PROFILE-REVIEW-RUBRIC.md`
- proof taxonomy and anti-pattern sections within the above docs
- thin-profile handling policy
- rollout sequencing and phase gates

## 2) Changes Requiring Explicit Review Approval

Any of the following must be approved before implementation:

- new profile variants or profile-type divergence
- layout divergence from shared practitioner architecture
- proof-system redesign or repositioning
- CTA language model changes (tone/structure)
- portrait treatment changes
- spacing system/hierarchy token changes
- recognition logic/ranking-order rule changes
- mobile composition changes affecting first-screen trust flow

## 3) Changes Allowed Without Governance Escalation

The following can proceed through normal editorial workflow:

- factual updates (role, office, mandates, credentials)
- typo and grammar fixes
- publication additions/removals
- representative matter updates (compliance-safe)
- minor copy refinements consistent with doctrine
- link repairs and source URL updates

Constraint: these changes must not violate anti-pattern rules.

## 4) Anti-Drift Triggers (Escalate Immediately)

If any trigger appears, pause and review:

- decorative proof treatments
- excessive whitespace used as authority signal
- luxury editorial minimalism behavior
- profile inconsistency in hierarchy/seriousness signals
- unsupported prestige language
- custom one-off profile layouts
- marketing-led "make it pop" requests
- SEO filler that degrades institutional tone

## 5) Review Cadence

- **Cohort gate reviews:** before each rollout cohort starts
- **Quarterly calibration:** doctrine, rubric, and anti-pattern refresh
- **Post-cohort review:** variance and quality delta assessment

Cadence objective: keep institutional calibration active and measurable.

## 6) Rollback Principles

- Ship in small reversible increments.
- Each change set must have a documented rollback path.
- If trust score, consistency, or tone discipline worsens, revert immediately.
- Prefer rollback over ad hoc patching when doctrine is violated.

## 7) Benchmark Isolation Policy

- Benchmark profiles remain isolated from live practitioner systems.
- No benchmark-only composition logic may leak into live pages without approval.
- Hidden benchmark routes stay noindex, sitemap-excluded, and directory-excluded.
- Benchmark findings inform rollout only through approved phase plans.

## 8) Institutional Consistency Requirements

All live practitioner profiles must preserve:

- shared trust language
- shared hierarchy logic
- shared informational pacing
- shared seriousness signals
- plain proof behavior
- restrained contact pathways

Any proposal improving one profile at the expense of system consistency should be rejected or deferred.
