# Bio Improvement Principles

Source basis: `LIVE-REFERENCE-AUDITS.md` and `SCHILLINGS-BIO-GAP-AUDIT.md`.  
Purpose: define durable principles for future practitioner bio improvement decisions.

## Non-Negotiable Direction

Future improvements should prefer:

- institutional consistency
- trust accumulation
- density without clutter
- plain proof systems
- practitioner credibility
- matter relevance
- restrained contact pathways
- shared scalable patterns

Future improvements should avoid:

- one-off art direction
- excessive whitespace
- luxury editorial minimalism
- profile-card aesthetics as authority proxy
- badges used as decoration
- over-designed proof treatments
- hidden or buried authority signals

## Principle 1: Shared System First

All practitioner profiles should remain on one shared architecture.

- No person-specific layout branches.
- No bespoke visual treatment for individual profiles.
- Variation should come from facts, remit, and evidence depth only.

Why: consistency itself is a trust signal in elite advisory contexts.

## Principle 2: Above-the-Fold Must Establish Authority Early

Above the fold should answer: who this practitioner is, what they do, and why this is credible.

Required behavior:

- Clear identity: name + role + office/practice context.
- Immediate remit clarity: scope before narrative.
- At least one early independent authority signal (where available), not buried below long prose.

Avoid:

- Opening with abstract prestige language.
- Deferring all proof to lower-page sections.

## Principle 3: Recognition Should Behave as Evidence, Not Decoration

Recognitions should support claims, not function as visual ornament.

Required behavior:

- Use recognitions as plain, scannable evidence.
- Keep relevance explicit (what signal this recognition supports).
- Maintain restrained volume and hierarchy (quality over quantity).

Avoid:

- Badge stacking without context.
- Highly stylized recognition treatments that outrank core biography.

## Principle 4: Portraits Should Support Identity, Not Carry Authority

Portrait treatment should confirm identity and professionalism while evidence carries authority.

Required behavior:

- Stable portrait-text hierarchy across profiles.
- Portrait prominence should not exceed proof and role signals.
- Keep portrait behavior consistent across profile types.

Avoid:

- Heroic portrait-led authority framing.
- profile-specific art-direction experiments.

## Principle 5: Biographies Should Accumulate Authority in Sequence

Bio writing should follow a repeatable authority arc.

Recommended paragraph logic:

1. current remit and evaluator relevance  
2. operating context and complexity  
3. proof-backed credibility (recognition, appointments, representative work, outputs)

Writing standards:

- specificity over adjectives
- concrete context over general prestige claims
- each paragraph does distinct work

Avoid:

- repeated assertions without new evidence
- long blocks that blur function

## Principle 6: Matter and Work Evidence Should Be Concrete but Controlled

Profiles should demonstrate real operating experience without breaching confidentiality.

Required behavior:

- include representative matter texture where appropriate
- use context descriptors (type, complexity, jurisdiction, stakeholder context)
- keep claims compliance-safe and proportionate

Avoid:

- vague "high-profile/high-stakes" repetition with no factual grounding
- over-disclosure of sensitive details

## Principle 7: Expertise and Context Should Improve Orientation

Practice/expertise context should help evaluators quickly navigate relevance.

Required behavior:

- expertise signals should be clear and plain-language where possible
- related context sections should support confidence, not create noise
- thin profiles should not lose all authority scaffolding

Avoid:

- taxonomy overload that obscures practitioner-level relevance
- context links that feel generic or disconnected from remit

## Principle 8: Contact Pathways Should Be Restrained and Procedural

Contact behavior should signal professionalism, discretion, and calm confidence.

Required behavior:

- neutral, procedural wording
- separation between authority claims and call-to-action language
- clear routing expectations

Avoid:

- urgency or pressure-based CTA language
- sales framing in practitioner profiles

## Principle 9: Information Density Should Be High-Signal, Low-Noise

Strong profiles are information-rich but structurally predictable.

Required behavior:

- stable sequence of identity, remit, proof, context, contact
- concise high-value facts in each section
- readable segmentation for scan and deep-read modes

Avoid:

- sparse minimalism that withholds key authority signals
- dense blocks without ordering cues

## Principle 10: Institutional Tone Should Be Calm, Precise, and Verifiable

Tone should project operational seriousness through composure and specificity.

Required behavior:

- measured language
- factual confidence
- claims that can be traced to evidence

Avoid:

- rhetorical inflation
- personality-led brand voice drift

## Cross-Profile Consistency Requirements

The following should remain consistent across all practitioner profiles:

- shared layout architecture and section order
- identity-first first-screen behavior
- portrait as supporting element
- plain proof system conventions
- restrained contact structure
- institutional tone baseline
- repeatable information hierarchy for mobile and desktop

## Quality Gate for Future Changes

Before any future implementation, each proposal should pass:

1. Does this improve trust accumulation in first-screen and skim paths?
2. Does it increase proof clarity without adding visual noise?
3. Does it preserve shared architecture and avoid one-off exceptions?
4. Does it improve practitioner credibility through concrete evidence?
5. Is it scalable across legal, communications, intelligence/security, and thin profiles?
6. Does it keep contact pathways restrained and institutional?

If not, it should not proceed.
