# Doctrine V1 Lock

Status: **Version 1 frozen**

Purpose: protect institutional consistency and prevent drift during Phase 1 rollout.

## Freeze Rules

- No casual rewrites.
- No unilateral doctrine edits during active cohort rollout.
- Any doctrine update requires explicit review gate approval and rationale.

## Doctrine Change Gate (When Changes Are Allowed)

Doctrine updates are permitted only when at least one applies:

- repeated cohort evidence shows doctrine ambiguity or failure
- anti-pattern gap appears not covered by current rules
- rubric produces inconsistent reviewer outcomes
- legal/compliance requires policy-level adjustment

Required for doctrine change:

1. change request summary
2. reason and evidence
3. impacted artifacts list
4. rollback path
5. approval sign-off

## What Is Not Frozen

The freeze does not block:

- factual profile updates
- typo corrections
- publication/matter updates
- approved Phase 1 implementation tasks within existing doctrine

## Versioning Convention

- Current lock: `Doctrine v1`
- Any approved update increments version (`v1.1`, `v2`, etc.)
- Every version change must include:
  - date
  - approved by
  - summary of changes
  - rationale

## Institutional Objective

This lock exists to preserve:

- trust mechanics
- evidence proximity discipline
- institutional tone consistency
- scalable publishing behavior over one-off preference changes
